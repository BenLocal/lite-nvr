# GB28181 P1-3 manual smoke test

Prereqs: a real GB28181 camera/NVR (or a software UAC), ZLM + nvr running with:
`NVR_GB_ENABLE=1 NVR_GB_SIP_ID=<20-digit> NVR_GB_DOMAIN=<realm> NVR_GB_PORT=5060 \
 NVR_GB_MEDIA_IP=<host-ip> [NVR_GB_PASSWORD=<pw>]`

1. **Register:** point the device's SIP platform config at `<host-ip>:5060` with the
   configured id/domain/password. Confirm nvr logs `gb28181: device registered: <id>`.
2. **Picker:** in the dashboard, add a device of type 国标 GB28181; the 国标设备
   dropdown lists the registered device; selecting it loads its channels via Catalog.
   Save with `{device_id, channel_id}`.
3. **On-demand pull:** open the device's live view. nvr should log
   `gb28181: pulling <id> channel <ch> -> stream <deviceId> (port N)`, the device
   starts sending PS/RTP to port N, and video plays.
   - **Confirm the playable app:** the stream is served at
     `http://<host>:8553/rtp/<deviceId>.live.flv`. If ZLM actually publishes the
     RtpServer stream under a different app, adjust `build_gb_flv_url` (init/device.rs)
     and the dashboard live-view URL accordingly. (Matching-by-stream-id means the
     pull itself is app-independent.)
4. **Teardown:** close all viewers; after ZLM's no-reader delay nvr logs
   `gb28181: released stream <deviceId>` and the device stops sending.
5. **Remove:** delete the device; confirm the mapping is gone (re-opening does not pull).
