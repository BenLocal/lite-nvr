# GB28181 crate design (`crates/gb28181`)

Date: 2026-07-01
Status: Approved (design), pending implementation plan

## 1. Goal

Add GB/T 28181 (Chinese national video-surveillance standard) support to lite-nvr.
The signaling is implemented as a **standalone, reusable Rust crate**
(`crates/gb28181`) built on top of [`rsipstack`](https://github.com/restsend/rsipstack),
supporting **both the server (上级平台/SIP UAS) and client (下级设备/网关, SIP UAC) roles**.

lite-nvr's own use of the crate is as a **GB28181 上级平台 (SIP server)**: national-standard
cameras / lower NVRs / lower platforms REGISTER to lite-nvr; lite-nvr sends INVITE to pull
their PS/RTP streams. The client role is delivered as a crate capability (for future cascade
to a superior platform) but is not integrated into nvr in the initial phases.

## 2. Scope

Full platform feature set, delivered in phases:

- **P1 — Live**: REGISTER + Keepalive + Catalog + real-time Play (INVITE → RTP receive → ZLM) + BYE.
- **P2 — PTZ**: DeviceControl (pan/tilt/zoom/preset).
- **P3 — Playback**: RecordInfo query + Playback INVITE + INFO (seek / scale) + TCP media.

The `client` facade (register up, answer Catalog/INVITE, hand off send) is built in the crate
and covered by loopback tests in parallel; nvr does not consume it yet.

## 3. Design decisions (the design tree)

Each row is a resolved branch from the design interview.

| # | Branch | Decision |
|---|--------|----------|
| 1 | Responsibility boundary | **Signaling only, media-agnostic.** The crate does all SIP + MANSCDP XML + SDP and never touches RTP bytes. Media (PS/RTP receive/send) is the caller's job, wired to ZLM via `rszlm` (`RtpServer` / `startSendRtp`). |
| 2 | client/server structure | **Shared `core` + two role facades.** `core` = rsipstack endpoint wrapper, MANSCDP codec, GB SDP, digest, GB-code types, dialog/session helpers. `server` (UAS) and `client` (UAC) reuse it. Enables an in-process client↔server loopback test. |
| 3 | Interaction model | **crate owns the event pump; async outbound methods + a single inbound event stream.** Facade exposes `async` methods (`register`, `catalog_query`, `invite_play`, `device_control`, `playback`, …) and `events() -> Receiver<GbEvent>` (device Registered/Online/Offline, Keepalive, CatalogItem, [client] InviteReceived, SessionClosed, …). |
| 4 | Wire model | **Strong-typed MANSCDP, lenient decode / strict encode.** One Rust type per message via `quick-xml` + serde. Decode tolerates missing fields, unknown elements, case variance, and **GB2312/GBK encodings** (transcode via `encoding_rs` per the XML declaration). GB-dialect SDP has its own small builder/parser (`s=Play/Playback`, `y=` SSRC, `f=`, `u=`). |
| 5 | Session lifecycle | **Registrar table + RAII media-session handle.** Server keeps an in-memory registrar (device SIP id → contact/transport/expiry/last-keepalive/online), refreshed by REGISTER + Keepalive, swept on expiry, surfaced via events and `server.devices()`. A media stream is an owned `MediaSession`: `stop().await` sends BYE, `Drop` enqueues a best-effort BYE, device-initiated BYE surfaces as `SessionClosed`. |
| 6 | Auth & credentials | **`AuthConfig` = `Open` \| `Shared(pw)` \| `Provider(closure)`.** digest challenge/verify, nonce generation/expiry, replay protection all live in the crate; the **password is supplied by the consumer** via a **synchronous** `Fn(&device_id) -> Option<password>` closure (nvr looks it up from its own store). Honor `Expires` (0 = unregister); client auto-answers 401 and refreshes early. |
| 7 | Media plane | **Handoff is pure data through the async/event API — no `MediaBackend` trait.** Server receive: `invite_play(target, MediaSpec{recv_addr, ssrc, transport, stream_type, codecs})` — caller has already opened its receiver. Client send: `InviteReceived{negotiation, remote_spec}` → caller opens sender → `negotiation.answer(local_spec).await`. **Transports (v1): UDP + TCP-passive + TCP-active** (`transport = Udp \| TcpPassive \| TcpActive`). SIP transport: UDP:5060 default, TCP optional (rsipstack-transparent). |
| 8 | Identity & config | **GB codes are opaque strings by default** (never reject a registration on a malformed code); optional `GbCode::parse()` helper. **The crate auto-generates GB-format SSRC** (live `0`/playback `1` + 5-digit domain-from-platform-id + 4-digit sequence). Config via builders (`GbServerConfig` / `GbClientConfig`): local SIP id, domain, listen addr:port, transport, auth, timers. |
| 9 | Resilience | **Lenient + partial results.** Catalog & other multi-packet replies aggregate by `SN` until `SumNum` or timeout, then **return the partial set + an `incomplete` flag**. Outbound queries auto-retry once. Keepalive miss ×3 → `Offline`. Unified `GbError` (Timeout / DeviceOffline / Sip / XmlDecode / Auth / Negotiation …). All thresholds configurable. |
| 10 | Testing | **Three layers.** (1) Unit: MANSCDP codec incl. **GB2312 samples**, GB SDP, SSRC, digest, registrar expiry. (2) **client↔server loopback integration** on UDP loopback exercising REGISTER → Catalog → INVITE → answer → BYE, asserting both sides' negotiated `MediaSpec` match (no real RTP). (3) Sample-replay decode hardening from real captures when available. |
| 11 | lite-nvr integration | **Dedicated on-demand `gb_bridge` in nvr.** Holds the global `GbServer` + a `stream_id ↔ (device_id, channel_id)` map + ZLM hooks: `on_media_not_found` → open `RtpServer` + `invite_play` → hold the `MediaSession`; `on_media_no_reader` → BYE + close. `input_type="gb28181"` devices only **register their mapping** in `ensure_device_pipe` (no always-on pipe). Dashboard adds a "国标 GB28181" input type with `{device_id, channel_id}` and a "pick from registered devices" picker (nvr API over the registrar/Catalog). |

## 4. Crate structure

```
crates/gb28181/
  src/
    lib.rs
    core/
      endpoint.rs     # rsipstack Endpoint wrapper + event pump task lifecycle
      manscdp.rs       # typed MANSCDP messages + quick-xml/serde codec (lenient decode)
      encoding.rs      # GB2312/GBK ↔ UTF-8 via encoding_rs, driven by the XML declaration
      sdp.rs           # GB-dialect SDP builder/parser (s=, y=ssrc, f=, u=)
      auth.rs          # digest challenge/verify, nonce store, AuthConfig
      gbcode.rs        # opaque GB code newtype + optional parser + SSRC generator
      session.rs       # MediaSession (RAII), dialog helpers, MediaSpec
      registrar.rs     # registered-device table + expiry sweep
      error.rs         # GbError
      event.rs         # GbEvent
    server.rs          # GbServer facade (UAS)
    client.rs          # GbClient facade (UAC)
  tests/
    loopback.rs        # client↔server end-to-end signaling
```

nvr side:
```
nvr/src/gb/
  bridge.rs            # global GbServer + stream map + ZLM on-demand hooks + MediaSession store
  config.rs            # GbServerConfig from NvrConfig
  api.rs               # list registered devices/channels for the dashboard picker
```
`nvr/src/init/device.rs` — `input_type == "gb28181"` registers the stream mapping with `gb_bridge`.

## 5. Key contracts

### 5.1 MediaSpec (media handoff, pure data)
```
MediaSpec {
  ssrc: u32,                 // GB 10-digit, crate-allocated unless caller-provided
  transport: Udp | TcpPassive | TcpActive,
  recv_addr | send_addr: SocketAddr,   // per direction
  stream_type: Play | Playback{range} | Download,
  codecs: [Ps(96/98) | H264(96) | ...],
  negotiated_remote: Option<SocketAddr>,  // set after 200 OK; required for TcpActive connect
}
```
- **UDP / TcpPassive**: caller opens the receiver first, then `invite_play`.
- **TcpActive**: `invite_play` → 200 OK carries the device's listen addr → the session exposes
  `negotiated_remote` → caller calls `RtpServer.connect(remote)`.

### 5.2 Server facade (nvr consumes)
```
GbServer::bind(GbServerConfig) -> GbServer            // starts the event pump
  .events() -> Receiver<GbEvent>
  .devices() -> Vec<RegisteredDevice>
  .catalog_query(device_id) -> Result<Catalog>       // aggregated, partial-tolerant
  .invite_play(target, MediaSpec) -> Result<MediaSession>
  .device_control(target, PtzCommand) -> Result<()>          // P2
  .playback(target, MediaSpec, time_range) -> Result<MediaSession>  // P3
```

### 5.3 Dependencies
`rsipstack`, `quick-xml`, `encoding_rs`, `serde`, `tokio`, `tokio-util`, `md-5`/`sha` (digest),
`anyhow`/`thiserror`, `bytes`. No media/RTP crate (media-agnostic).

## 6. Out of scope (initial)

- PS/RTP packetization (delegated to ZLM).
- Alarm subscribe/notify, mobile position (GPS), voice broadcast / two-way audio.
- nvr integration of the `client`/cascade role (crate capability only, loopback-tested).
- TLS/WS SIP transport (rsipstack supports it; not exercised initially).

## 7. Phasing summary

- **P1**: `core` + `server` (REGISTER/Keepalive/Catalog/Play/BYE, UDP+TCP receive) + `gb_bridge`
  + on-demand hooks + `input_type=gb28181` + dashboard picker + loopback test.
- **P2**: DeviceControl / PTZ (crate + API + frontend control).
- **P3**: RecordInfo + Playback INVITE + INFO seek/scale + frontend playback.
- **Parallel**: `client` facade in the crate, covered by loopback tests (no nvr integration yet).
